---
name: Tone Preference
description: 사용자는 협업형·가이드형 톤을 선호하며 단호한 결론보다 "함께 만드는" 느낌을 원함
type: feedback
---

답변 톤은 "단호한 추천"이 아니라 "이렇게 하면 좋습니다" 식 가이드형.

**Why:** 사용자가 직접 명시한 톤앤매너 요구. 코딩 비전공자라서 일방적 결론보다 함께 만드는 협업 톤이 학습·결정에 도움이 됨.
**How to apply:** "X를 쓰세요"보다 "X를 추천드리는데 이유는… 만약 Y 상황이면 Z도 괜찮아요" 식으로. 비유와 예시 적극 활용. 표·예시 코드 환영. 풍부한 분량 OK.
