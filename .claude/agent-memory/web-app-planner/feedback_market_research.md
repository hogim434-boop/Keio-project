컨셉을 "이건 없는 서비스다"라고 추천할 때는 반드시 실제 WebSearch/WebFetch로 한국 시장을 검색하고, 검색 쿼리와 결과 링크를 답변에 명시해야 함.

**Why:** 사용자가 "추측이 아니라 검색 결과 기반으로", "사용자 신뢰의 근거"라고 명시적으로 요구. 코딩 비전공자 사용자가 직접 시장 검증을 할 수 없으므로 기획자(Claude)가 검증 책임을 짐.

**How to apply:**
- 영역별 대표 서비스 2~3개를 검색으로 확인할 것
- 검색해도 안 나오는 컨셉만 "빈틈"으로 추천 가능
- 검색 쿼리 자체를 답변에 노출 (예: "'무로그인 메모 웹앱 2026 한국'으로 검색 시 0건")
- 0건이라고 단정하지 말고 "검색 상위에 안 잡힘 = 마케팅 안 된 = 실질적 빈틈" 정도로 신중히 표현
