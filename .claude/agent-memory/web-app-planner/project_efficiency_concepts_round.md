2026-04-30 라운드에서 1회성·자랑형 가정을 완전히 폐기하고 반복 사용 효율 도구로 5개 컨셉 제시.

**시장 빈틈 (검색으로 확인된 것)**:
- 약속·일정 조율: 모두의시간/언제만나는 시간만, 장소·정산은 분리
- 정산: 카카오페이는 단발성, 누적·관계형 정산 빈틈
- 구독 관리: 토스/뱅크샐러드는 무거움, 피클플러스/키노라이츠는 다른 목적 — 무로그인 좁은 용도 빈틈
- 링크 큐레이션: 노션은 무겁고 카톡 "나에게 보내기"는 검색 안 됨
**빈틈이 좁아 제외한 영역**: 학교/과제(에브리타임·유니버), 냉장고(유통기한 언제지·스마트 냉장고), 약/영양제(테이크·오늘의약)

**TOP 2 컨셉**:
1. 구독 정리장 (Sub-Board) — 무로그인 로컬스토리지, 매달 결제일 트리거 재방문, 구현 가장 쉬움(★★)
2. 누적 더치페이 (Tab) — 룸메이트·커플 페르소나, 매일 재방문 가능성 가장 강함, 구현 ★★★

**Why:** 사용자가 "재방문성 + 실제 효용 + UX 단순함"을 평가축으로 명시. 구독 정리장은 구현 부담이 가장 낮으면서 매달 자동 재방문 트리거 보유 — 코딩 초보자에게 현실적.

**How to apply:**
- 다음 라운드는 사용자가 1·2순위 중 하나 고른 뒤 PRD/와이어프레임/MVP 우선순위로 진입
- 다른 영역(학교/냉장고/약 등) 다시 추천하지 말 것 — 이미 검색으로 강자 확인됨
- 컨셉 A(모임 한 페이지)·C(여행 가계부)·D(보낼 링크 보드)는 사용자가 명시적으로 다시 꺼내지 않으면 깊이 안 들어감
