2026-04-30 라운드에서 사용자가 방향 전환. "별의 좌표(Star Deed)"와 테스트형/자랑형 카테고리 폐기. 새 방향: "사람들이 더 효율적인 삶을 살기 위해 쓸 수 있는 웹앱" + 현존 서비스에 없는 차별화.

같은 날 후속 라운드에서 사용자가 **"1회성 + 자랑형" 모델 자체를 메모리에서 지우라고 명시 요청.** 이제 사용자 선호 모델은 무로그인·무광고·인스타/카톡 공유 채널까지만 유효하고, "1회성 폭발 + 결과 카드 자랑형" 가정은 사라짐.

**Why:** 사용자가 의미·실용성을 우선하는 방향으로 분명히 선회함. 자랑 동기 중심 설계는 더 이상 사용자가 원하는 게 아님.

**How to apply:**
- 효율형/실용형 웹앱은 본래 반복 사용 모델임을 자연스럽게 받아들이고, "1회성 폭발"로 억지로 끼워 맞추지 말 것
- 더 이상 "선택지 A(효율 + 결산 카드 자랑) vs B(효율 가장한 1회성 자랑 도구)" 두 갈래 프레이밍을 사용하지 말 것 — B 옵션은 폐기
- 시장 조사는 반드시 실제 WebSearch 수행. 머릿속 지식만으로 "없다"고 단정 금지
- 자랑·과시 동기보다 실제 사용자 효용·재방문성·UX 단순함을 우선 평가축으로 삼을 것
