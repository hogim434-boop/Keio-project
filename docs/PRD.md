# Jukulog (塾ログ) MVP PRD — 자유 게시판 버전

## 핵심 정보

**목적**: 게이오 대학 재학생만 입장 가능한 클로즈드 익명 자유 게시판을 제공하여, 재학생들이 캠퍼스 생활·연애·취업·일상 등 모든 소재를 자유롭게 이야기할 수 있는 안전한 공간을 만든다.
**사용자**: 게이오 대학교 재학생 (학부생 및 대학원생, keio.jp 이메일 보유자 한정)

> **피벗 배경**: 강의 리뷰 서비스(Jukulog)는 PENMARK(학부생 80% 점유)와의 차별화가 어렵다고 판단. "keio.jp 만 들어올 수 있는 클로즈드 자유 게시판" — 한국의 디시인사이드(DCInside) 성향의 갤러리/게시판 구조 — 으로 전환.

## 사용자 여정

```
1. 랜딩 페이지 (비로그인)
   ↓ "지금 시작하기" 또는 "로그인" 클릭

2. 회원가입 페이지
   → keio.jp 이메일 입력 (타 도메인 즉시 차단)
   → 비밀번호 입력 → 커뮤니티 가이드라인 동의 체크박스
   → "가입하기" 버튼 → Supabase 이메일 인증 메일 발송
   ↓ 인증 메일 클릭 → auth/callback 처리

3. 닉네임·프로필 설정 페이지 (signup/setup)
   → 닉네임 / 소속 캠퍼스 / 학부 입력
   ↓ "완료" 버튼 → 자동 리디렉션

4. 게시판 홈 (피드) 페이지 — 첫 진입
   ┌─────────────────────────────────┐
   │ 🔥 지금 핫한 글 (가로 스와이프)  │ ← F014: 24시간 추천 TOP 3
   ├─────────────────────────────────┤
   │ 갤러리 아이콘 행 (가로 스크롤)   │ ← F015: 원형 아이콘 + 라벨
   ├─────────────────────────────────┤
   │ 정렬 토글 (최신글 / 인기글)      │ ← sticky
   ├─────────────────────────────────┤
   │ 게시글 카드 (본문 미리보기 +     │ ← F018: 인라인 ❤️/🔖
   │  인라인 추천·북마크)             │
   └─────────────────────────────────┘
   ↓ 갤러리 아이콘 클릭 → 해당 카테고리 필터 적용
   ↓ 카드 클릭 → 게시글 상세 / 카드 ❤️·🔖 클릭 → 인라인 반응
   ↓ 핫 글 캐러셀 클릭 → 게시글 상세

   [글 읽기] → 게시글 상세 페이지
   → 댓글 작성 or 추천/비추천 or 북마크
   → 신고 (문제 게시글 발견 시)

   [글 쓰기] — 어느 페이지에서든 중앙 FAB 클릭 (F016)
   → Bottom Sheet 80vh 슬라이드 업 (F017)
   → 카테고리 선택 + 제목 + 본문 입력
   → 익명/닉네임 토글 → "등록" 버튼
   → 작성 중 닫기 시 localStorage 자동 임시 저장
   ↓ 등록 완료 → 방금 쓴 게시글 상세 페이지

5. 탐색·인기 페이지
   → 인기 게시글 / 키워드 검색 (검색 결과 카드도 미리보기 + 인라인 반응)
   ↓ 결과 클릭 → 게시글 상세 페이지

6. 마이페이지
   → 내 글 / 내 댓글 / 내 북마크 목록 확인
   → 내 글 삭제
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|--------------|------------|
| **F001** | keio.jp 이메일 인증 | keio.jp / g.keio.ac.jp / sfc.keio.ac.jp 도메인만 가입 허용. 타 도메인은 폼 단계에서 즉시 차단. 서버 사이드에서도 재검증 | 외부인 차단 — 클로즈드 커뮤니티의 핵심 진입장벽 | 회원가입 페이지 |
| **F002** | 카테고리(갤러리) 분류 | 주제별 갤러리 + 캠퍼스별 갤러리 혼합 구조. 게시글마다 카테고리 1개 필수 선택 | 게시판 구조의 근간 — 사용자가 원하는 주제 글만 필터링 | 게시판 홈, 게시글 작성 페이지, 게시글 상세 페이지 |
| **F003** | 게시글 작성 | 제목(최대 100자) + 본문(최대 5,000자, 최소 10자) + 카테고리 선택 + 익명/닉네임 선택. 이미지 첨부는 MVP 이후로 보류 | 커뮤니티의 핵심 콘텐츠 생산 기능 | 게시글 작성 페이지 |
| **F004** | 게시글 목록 조회 | 카테고리별 피드 + 전체/최신/인기 정렬 탭 + 무한 스크롤 | 커뮤니티 진입 후 콘텐츠를 탐색하는 메인 화면 | 게시판 홈 |
| **F005** | 게시글 상세 + 댓글 | 본문 전체 표시 + 댓글 작성/삭제 + 대댓글(1단계). 댓글도 익명/닉네임 선택 가능 | 글 읽기와 토론이 일어나는 핵심 상호작용 페이지 | 게시글 상세 페이지 |
| **F006** | 추천/비추천 | 게시글·댓글에 추천 또는 비추천 1회. 인기 정렬의 기준 데이터 | 인기글 선별 + 사용자 반응 표현 수단 | 게시글 상세 페이지, 게시판 홈 |
| **F007** | 검색 | 제목/본문 키워드 검색 (PostgreSQL pg_trgm 활용). 검색어 없을 때는 인기 게시글 노출 | 특정 주제를 찾는 사용자의 직접 탐색 경로 | 탐색·인기 페이지 |
| **F008** | 신고 기능 | 게시글/댓글에 신고 버튼. 신고 유형 선택(욕설/명예훼손/스팸/불법) + 선택적 보충 설명 → 어드민 검토 큐로 이동 | 자유 게시판 특성상 명예훼손·욕설 리스크 매우 높음 — MVP 필수 안전장치 | 게시글 상세 페이지, 어드민 페이지 |
| **F009** | 북마크(스크랩) | 게시글을 북마크 저장. 마이페이지에서 목록 확인 | 나중에 다시 읽고 싶은 글 저장 수단 | 게시글 상세 페이지, 마이페이지 |
| **F014** | 핫 피드 캐러셀 | 게시판 홈 최상단에 "지금 뜨는 글 TOP 3" 가로 스와이프 캐러셀 (최근 24시간 추천 수 기준) | 첫 진입 시 "뭐가 핫한지" 즉시 인지 — 이탈률 감소, 추천 시스템 가시화 | 게시판 홈 |
| **F015** | 갤러리 아이콘 행 | 카테고리를 인스타 스토리 식 원형 아이콘 + 라벨로 가로 스크롤 노출 | 칩보다 시각적 인지도 높음 — 학생이 원하는 갤러리를 빠르게 찾는 진입점 | 게시판 홈 |

| **F017** | 게시글 작성 Bottom Sheet | FAB 또는 글쓰기 진입 시 화면 하단에서 80vh Sheet 슬라이드 업. 작성 중 내용은 localStorage 자동 임시 저장 | 페이지 전환의 무게 제거. 점심시간 5분 짧은 글 작성 친화적 | 모든 페이지 (FAB 트리거) |
| **F018** | 카드 미리보기 + 인라인 반응 | 게시글 카드에 본문 첫 80~100자 미리보기 + 추천·북마크 버튼을 카드에서 직접 토글 가능 | 클릭 전 정보 풍부 → 피드 만족도 향상. 가벼운 반응 증가 → 추천 데이터 품질 향상 | 게시판 홈, 탐색·인기 페이지 |

## 디자인 원칙

### 모바일 우선 (Mobile First)

대학생의 주 사용 환경이 스마트폰임을 전제로, 모바일 뷰를 기준으로 설계한다.

| 항목 | 기준값 |
|------|--------|
| 기준 뷰포트 | 390px (iPhone 14 기준) |
| 최소 터치 영역 | 44×44px (버튼, 탭 아이콘) |
| 콘텐츠 여백 | 좌우 16px |
| 하단 안전 영역 | Safe Area Inset 대응 (`pb-safe`) |
| 데스크탑 대응 | 최대 너비 768px로 중앙 정렬 (tablet 이상) |

- 모든 인터랙션 요소(버튼, 탭, 카드)는 엄지손가락 닿기 편한 하단 영역 우선 배치
- 스크롤은 세로 단방향. 가로 스크롤은 카테고리 탭 칩에만 허용
- 폰트 최소 크기 14px, 주요 텍스트 16px 이상
- 게시글 카드는 제목 + 카테고리 + 작성자(익명 or 닉네임) + 추천 수 + 댓글 수 + 시간만 표시 (간결하게)

## 메뉴 구조

```
비로그인 — 상단 헤더 (간소)
├── 로고 (랜딩 페이지로 이동)
├── 로그인 버튼
└── 회원가입 버튼

로그인 후 — 하단 탭 네비게이션 (3탭 + 중앙 부유 FAB)
├── [탭 1] 게시판  (Home 아이콘)        - F002, F004, F006, F014, F015, F018
│
├── ●●● 중앙 FAB (PenSquare 원형 버튼)  - F003, F016, F017
│        ↑ 탭 바보다 한 단 위로 떠 있는 부유 버튼
│        ↑ 모든 로그인 페이지에서 노출
│        ↑ 누르면 글쓰기 Bottom Sheet 슬라이드 업
│
├── [탭 2] 탐색    (Search 아이콘)      - F007, F018
└── [탭 3] 마이    (User 아이콘)        - F009, F011

  ※ 상단 헤더는 현재 페이지 제목 + 필요 시 뒤로가기 버튼만 표시
  ※ 로그아웃은 마이페이지 탭 내부에 위치
  ※ 신고는 게시글/댓글의 점 3개(…) 메뉴 → 신고 선택으로 진입
  ※ 글쓰기는 별도 라우트 페이지 없이 FAB → Bottom Sheet 로 처리 (F017)

어드민 전용 — 별도 접근 (/admin)
└── 신고 검토 큐 관리 - F012
```

### 하단 탭 네비게이션 상세

| 위치 | 아이콘 | 레이블 | 기능 ID |
|------|--------|--------|---------|
| 좌측 탭 | `Home` | 게시판 | F002, F004, F006, F014, F015, F018 |
| 중앙 FAB | `PenSquare` | (라벨 없음, 원형 버튼) | F003, F016, F017 |
| 우측 탭 1 | `Search` | 탐색 | F007, F018 |
| 우측 탭 2 | `User` | 마이 | F009, F011 |

- 탭 높이: 56px + Safe Area
- 중앙 FAB: 56×56px 원형, primary 배경 + 흰색 아이콘, 탭 바 기준 위로 8px 부유, drop shadow 적용
- 활성 탭: 아이콘 + 레이블 강조 (primary 색상)
- 비활성 탭: 아이콘 + 레이블 muted 색상
- 하위 페이지(게시글 상세 등) 이동 시에도 탭 바 + 중앙 FAB 고정 표시
- FAB 누르면 어떤 페이지에서든 글쓰기 Bottom Sheet 슬라이드 업 (현재 페이지 컨텍스트 유지)

## 카테고리(갤러리) 구조

게시글은 반드시 카테고리 1개에 속해야 한다. 카테고리는 "주제별"과 "캠퍼스별" 두 가지 축으로 구성한다.

### 주제별 갤러리

| 카테고리 슬러그 | 표시 이름 | 설명 |
|----------------|-----------|------|
| `free` | 자유 | 무엇이든 OK — 분류하기 어려운 글 |
| `love` | 연애·썸 | 연애 고민, 고백 썰, 만남 후기 |
| `study` | 학업·수업 | 공부 방법, 수업 꿀팁, 시험 준비 |
| `job` | 취업·인턴 | 취업 준비, 인턴 후기, 자기소개서 |
| `daily` | 일상·잡담 | 학교 생활, 동아리, 먹방, 일상 이야기 |
| `question` | 질문·도움 | 뭐든 물어보세요 |

### 캠퍼스별 갤러리

| 카테고리 슬러그 | 표시 이름 | 대상 캠퍼스 |
|----------------|-----------|------------|
| `mita` | 미타 갤러리 | 미타 캠퍼스 (문·경·법) |
| `hiyoshi` | 히요시 갤러리 | 히요시 캠퍼스 (이·공·의·약·간호 1·2학년 등) |
| `sfc` | SFC 갤러리 | 쇼난 후지사와 캠퍼스 (총합정책·환경정보) |
| `yagami` | 야가미 갤러리 | 야가미 캠퍼스 (이공 3·4학년·대학원) |
| `shinanomachi` | 시나노마치 갤러리 | 시나노마치 캠퍼스 (의·간호) |
| `shiba` | 시바공립 갤러리 | 시바공원 지구 (약학·간호 일부) |

> **운영 지침**: 초기에는 자유·연애·학업·취업·일상·질문·미타·히요시·SFC 9개 카테고리로 시작. 사용자가 늘면 캠퍼스 갤러리 추가.

### 게시판 홈

> **구현 기능:** `F002`, `F004`, `F006`, `F014`, `F015`, `F018` | **인증:** 로그인 필수

| 항목 | 내용 |
|------|------|
| **역할** | 커뮤니티의 메인 피드. "지금 핫한 글" + "갤러리 진입" + "최신/인기 피드" 3층 구조 |
| **진입 경로** | 로그인 성공 후 자동 이동, **하단 탭 "게시판"** 탭 |
| **사용자 행동** | 핫 글 캐러셀 스와이프 → 갤러리 아이콘 선택 → 정렬 토글 → 카드 미리보기 확인 → 추천/북마크/상세 클릭 |
| **레이아웃 (위→아래)** | **1층 — 🔥 지금 핫한 글 (F014)**: 최근 24시간 추천 수 TOP 3 게시글을 가로 스와이프 캐러셀로 표시. 각 슬라이드에 카테고리 배지 + 제목 + 추천 수<br>**2층 — 갤러리 아이콘 행 (F015)**: 카테고리를 인스타 스토리 식 원형 아이콘 + 라벨로 가로 스크롤. (예: 🌸자유 / ❤️연애 / 📚학업 / 💼취업 / 🍱일상 / ❓질문 / 미타 / 히요시 / SFC). "전체"는 첫 항목으로 항상 노출<br>**3층 — 정렬 토글**: 세그먼트 컨트롤 (최신글 / 인기글). 인기글은 7일간 추천 수 기준<br>**4층 — 게시글 카드 무한 스크롤 (F018)**: 카테고리 배지, 작성자(익명 or 닉네임), 작성 시간, 제목, **본문 첫 80~100자 미리보기**, 인라인 ❤️추천·💬댓글수·🔖북마크 버튼 |
| **카드 인라인 반응** | 카드의 ❤️ 또는 🔖 버튼을 목록에서 직접 토글 (페이지 진입 없이 추천/북마크 가능) — F006, F009. 응답은 낙관적 업데이트 후 fetch |
| **모바일 UX** | 1·2층은 sticky 하지 않고 일반 흐름. 3층(정렬 토글)만 스크롤 시 sticky 처리. 카드 길게 누르면 컨텍스트 메뉴(공유·신고) — P1 |
| **다음 이동** | 핫 글 캐러셀 클릭 → 게시글 상세, 갤러리 아이콘 클릭 → 해당 카테고리만 필터된 피드, 카드 본문 영역 클릭 → 게시글 상세 |

## 데이터 모델

> **RLS(Row Level Security) 원칙**
> - 게시글/댓글/북마크/반응: 본인만 INSERT/UPDATE/DELETE 가능 (SELECT는 로그인 사용자 전체 허용)
> - **익명이라도 user_id는 반드시 DB에 저장** — 신고 추적 및 어드민 처리를 위해 필수
> - 신고 테이블: 신고자 user_id 저장, 어드민만 전체 READ 가능
> - 어드민 권한: profiles.role = 'admin' 으로 관리

### profiles (사용자 프로필) — 기존 유지

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | Supabase Auth UID와 동일 | UUID |
| nickname | 닉네임 (커뮤니티 표시명) | TEXT |
| campus | 소속 캠퍼스 | TEXT |
| department | 소속 학부/학과 | TEXT |
| role | 권한 구분 (user / admin) | TEXT |
| created_at | 가입일 | TIMESTAMP |

### posts (게시글) — 신규

| 필드 | 설명 | 타입/관계 |
|------|------|----------|
| id | 고유 식별자 | UUID |
| user_id | 작성자 (신고 추적용, 화면에는 미표시 가능) | → profiles.id |

## 기술 스택

### 프론트엔드 프레임워크

- **Next.js 16.2.4** (App Router) — React 풀스택 프레임워크 (현재 프로젝트 사용 버전)
- **TypeScript 5.x** (strict 모드) — 타입 안전성
- **React 19.2.4** — UI 라이브러리

### 스타일링 & UI

- **Tailwind CSS v4** (`@tailwindcss/postcss` 방식, 설정 파일 없음) — 유틸리티 CSS
- **shadcn/ui** (style: radix-nova) — 고품질 React 컴포넌트
- **Lucide React** — 아이콘 (`Home`, `Search`, `PenSquare`, `User` 등)
- **Framer Motion** — 게시글 카드 스태거 애니메이션, 바텀시트 슬라이드 등

### 폼 & 검증

- **React Hook Form 7.x** — 폼 상태 관리
- **Zod** — 스키마 검증 (keio.jp 이메일 도메인 검증, 게시글 길이 검증 포함)

### 백엔드 & 데이터베이스

- **Supabase Auth** — 이메일 인증, 세션 관리 (기존 유지)
- **Supabase Database** (PostgreSQL) — 게시판 전체 데이터
- **Supabase Row Level Security (RLS)** — 본인 글만 삭제, 익명 user_id 보호
- **PostgreSQL pg_trgm 확장** — 제목/본문 키워드 검색

### 배포 & 호스팅

- **Vercel** — Next.js 최적화 배포

### 패키지 관리

- **npm** — 의존성 관리

## 주요 리스크 및 대응

| 리스크 | 상세 내용 | 대응 방안 |
|--------|-----------|----------|
| **명예훼손·욕설 리스크** | 자유 게시판 특성상 일반 강의 리뷰보다 훨씬 높음. 특히 실명 언급, 성희롱, 학교 특정인 비방 | 가이드라인 동의 필수화(F013), 신고 큐(F008/F012) MVP 필수 포함, 어드민 48시간 내 처리 목표, soft delete로 이력 보존 |
| **콜드 스타트 문제** | 초기 글이 없으면 아무도 안 오고, 아무도 안 오니 글이 없는 악순환 | 어드민 계정으로 시드 게시글 작성, 지인 베타 그룹(20~30명) 먼저 초대, keio.jp SNS 채널 홍보 |
| **익명성 vs 책임성 균형** | 완전 익명이면 품질 저하, 실명이면 솔직함 감소 | keio.jp 인증으로 가입 장벽 유지(외부인 차단), DB에는 user_id 저장(신고 추적), 화면에만 익명 표시 |
| **모더레이션 비용** | 1인 운영자가 감당 가능한 수준인가 | MVP에서는 신고 기반 사후 대응(Pull 모델)만. 자동 필터·상시 모니터링은 Phase 2 이후. 신고 처리 UI를 최대한 단순하게 설계 |
| **keio.jp 이메일 우회** | 졸업생 이메일 또는 타인 이메일로 가입 시도 | Supabase Auth 이메일 인증 필수화 + 서버 사이드 도메인 재검증 + 문제 계정 어드민 강제 비활성화 |
