# Jukulog (塾ログ) MVP PRD

## 핵심 정보

**목적**: 게이오 대학 재학생이 수강신청 전 신뢰할 수 있는 강의 정보를 얻지 못하는 정보 비대칭 문제를 해결한다.
**사용자**: 게이오 대학교 재학생 (학부생 및 대학원생, keio.jp 이메일 보유자)

---

## 사용자 여정

```
1. 랜딩 페이지 (비로그인)
   ↓ "리뷰 보기" 또는 "시작하기" 클릭

2. 회원가입 페이지
   → keio.jp 이메일 입력 → 인증 메일 발송
   ↓ 이메일 인증 링크 클릭

3. 이메일 인증 완료
   ↓ 자동 리디렉션 → 강의 목록 페이지

4. 강의 목록 페이지 (로그인 상태)
   ↓ 캠퍼스/학부/학기 필터 선택 → 강의 카드 클릭

5. 강의 상세 페이지
   ↓ "리뷰 작성" 버튼 클릭 → 내가 수강한 강의인 경우

   [리뷰 없음] → 리뷰 작성 폼 표시 → 리뷰 제출 → 강의 상세 페이지
   [이미 작성] → 내 리뷰 수정 폼 표시 → 저장 → 강의 상세 페이지

6. 강의 검색 페이지
   ↓ 교수명 / 강의명 검색 → 결과 클릭 → 강의 상세 페이지

7. 마이페이지
   ↓ 내가 작성한 리뷰 목록 확인 → 수정/삭제 가능
```

## 기능 명세

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|---------------|------------|
| **F001** | keio.jp 이메일 인증 | 재학생 전용 접근 제어. keio.jp 도메인 이메일만 회원가입 허용 | 재학생 신뢰성 보장 — 외부인 유입 차단, 리뷰 신뢰도의 근간 | 회원가입 페이지 |
| **F002** | 5축 강의 리뷰 작성 | 종합 평점 + 출석, 시험/과제 난이도, 학점 취득 난이도, 강의 스타일 항목 포함 | 핵심 가치 — 경쟁 서비스와 차별화되는 다차원 정보 제공 | 강의 상세 페이지 |

| **F004** | 강의 검색 | 강의명, 교수명으로 전문 검색 | 직접 원하는 강의를 찾는 사용자 지원 | 강의 검색 페이지 |
| **F005** | 강의 상세 정보 열람 | 강의 기본 정보 + 집계 평점 + 리뷰 목록 표시 | 정보 소비의 핵심 페이지 — 서비스의 최종 가치 전달 지점 | 강의 상세 페이지 |
| **F006** | 익명 리뷰 | 작성자 닉네임 비공개 처리 (내부적으로 사용자 ID 저장, 화면에는 익명 표시) | 솔직한 리뷰 작성 유도 — 실명 노출 시 자기검열 우려 | 강의 상세 페이지, 리뷰 작성 폼 |
| **F007** | 리뷰 수정 및 삭제 | 작성자 본인만 자신의 리뷰를 수정하거나 삭제 가능 | 잘못된 정보 정정 권한 보장 | 마이페이지, 강의 상세 페이지 |

## 디자인 원칙

### 모바일 우선 (Mobile First)

대학생의 주 사용 환경이 스마트폰임을 전제로, 모바일 뷰를 기준으로 설계한다.

| 항목 | 기준값 |
|------|--------|
| 기준 뷰포트 | 390px (iPhone 14 기준) |
| 최소 터치 영역 | 44×44px (버튼, 탭 아이콘) |
| 콘텐츠 여백 | 좌우 16px |
| 하단 안전 영역 | Safe Area Inset 대응 (`pb-safe`) |
| 데스크탑 대응 | 최대 너비 768px로 중앙 정렬 (tablet 이상) |

- 모든 인터랙션 요소(버튼, 탭, 카드)는 엄지손가락 닿기 편한 하단 영역 우선 배치
- 스크롤은 세로 단방향. 가로 스크롤은 필터 칩에만 허용
- 폰트 최소 크기 14px, 주요 텍스트 16px 이상

## 메뉴 구조

```
비로그인 — 상단 헤더 (간소)
├── 로고 (랜딩 페이지로 이동)
├── 로그인 버튼
└── 회원가입 버튼

로그인 후 — 하단 탭 네비게이션 (Tab Bar)
├── [탭 1] 강의 목록  (Home 아이콘)    - F003
├── [탭 2] 검색       (Search 아이콘)  - F004
└── [탭 3] 마이페이지 (User 아이콘)    - F007, F011

  ※ 상단 헤더는 현재 페이지 제목 + 필요 시 뒤로가기 버튼만 표시
  ※ 로그아웃은 마이페이지 탭 내부에 위치
  ※ 프로필 설정은 마이페이지 탭에서 진입

어드민 전용 — 별도 URL (/admin)
└── 강의 관리 - F012
```

### 하단 탭 네비게이션 상세

| 탭 | 아이콘 | 레이블 | 이동 경로 |
|----|--------|--------|----------|
| 강의 목록 | `Home` | 강의 | `/courses` |
| 검색 | `Search` | 검색 | `/search` |
| 마이페이지 | `User` | 마이 | `/my` |

- 탭 높이: 56px + Safe Area
- 활성 탭: 아이콘 + 레이블 강조 (primary 색상)
- 비활성 탭: 아이콘 + 레이블 muted 색상
- 하위 페이지(강의 상세 등) 이동 시에도 탭 바는 고정 표시

### 강의 상세 페이지

| 항목 | 내용 |
|------|------|

| **다음 이동** | 리뷰 제출 성공 → Bottom Sheet 닫힘 + 리뷰 목록 갱신, 뒤로가기 → 강의 목록 페이지 |

### 마이페이지

> **구현 기능:** `F007`, `F011` | **인증:** 로그인 필수

| 항목 | 내용 |
|------|------|
| **역할** | 사용자가 작성한 리뷰를 한곳에서 관리하는 개인 공간 |
| **진입 경로** | **하단 탭 "마이" 탭 탭** |

## 기술 스택

### 프론트엔드 프레임워크

- **Next.js 16.2.4** (App Router) — React 풀스택 프레임워크 (현재 프로젝트 사용 버전)
- **TypeScript 5.x** (strict 모드) — 타입 안전성
- **React 19.2.4** — UI 라이브러리

### 스타일링 & UI

- **Tailwind CSS v4** (`@tailwindcss/postcss` 방식, 설정 파일 없음) — 유틸리티 CSS
- **shadcn/ui** (style: radix-nova) — 고품질 React 컴포넌트
- **Lucide React** — 아이콘

### 폼 & 검증

- **React Hook Form 7.x** — 폼 상태 관리
- **Zod** — 스키마 검증 (keio.jp 이메일 도메인 검증 포함)

### 백엔드 & 데이터베이스

- **Supabase Auth** — 이메일 인증, 세션 관리
- **Supabase Database** (PostgreSQL) — 강의 및 리뷰 데이터
- **Supabase Row Level Security (RLS)** — 본인 리뷰만 수정/삭제 가능 권한 제어
- **PostgreSQL 전문 검색 (pg_trgm)** — 강의명/교수명 검색

### 배포 & 호스팅

- **Vercel** — Next.js 최적화 배포

### 패키지 관리

- **npm** — 의존성 관리
